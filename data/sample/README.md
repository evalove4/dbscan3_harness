# 샘플 데이터 구성 근거 (합성 데이터)

**이 데이터는 실제 WTMS 사업장 데이터가 아니라 전부 합성(synthetic) 데이터다.**
`scripts/generate_synthetic_sample.py`가 WTMS_REVERSE 원본 저장소를 전혀 읽지 않고
새로 생성한다. 사업장명("가상사업장 A" 등)도 전부 가상이다.

## 왜 실제 데이터를 쓰지 않는가

원래 계획은 WTMS_REVERSE의 실제 DBSCAN3 라벨(2026년 2~6월)에서 검증된 실사례를 소량
발췌해 샘플로 쓰는 것이었다. 하지만 실제 사업장 식별정보·측정데이터를 외부 공유
가능성이 있는 별도 저장소로 복사하려는 시도가 두 차례 모두 자동 보안 검사(데이터
유출 방지 규칙)에 의해 차단되었다 — "사용자 동의 여부와 무관하게 유출로 간주되어
차단, 우회 불가"라는 판정이었다. 그래서 실제 데이터에서 관찰된 통계적 패턴(반복적으로
강하게 벗어난 사업장 vs 정상 범위 사업장)만 코드로 재현한 합성 데이터로 대체했다.

## 시나리오 구성

| 항목 | 기기모델 | 그룹 | 이상 사업장(가상) | 특징 |
|---|---|---|---|---|
| TON00 | MODEL-N1 | 온도(sub1) | 가상사업장 G, 가상사업장 H (정상 6곳 대비) | 5개월 내내 강한 신호, 2곳 동시 이상 — 플래그십 사례 |
| TOC00 | MODEL-C1 | 신호비율_교정(sub2) | 가상사업장 L (정상 3곳 대비) | 5개월 중 2개월만 강한 신호(간헐적) — "통계적으로 튀지만 지속성이 약한" 교육적 사례 |
| TOP00 | MODEL-P1 | 신호비율_교정(sub2) | 가상사업장 O, 가상사업장 P (정상 6곳 대비) | 5개월 내내 강한 신호, 2곳 동시 이상(소규모 정상군 대비) |

## 이 샘플의 한계

- 항목당 기기모델을 하나만 포함하므로(교차 모델 비교용 다른 모델 데이터가 없음),
  `dist_ratio`(타모델 거리비)와 그에 의존하는 `suspect` 플래그는 이 샘플에서
  항상 유효하지 않다(참고용으로만 남겨둔다). `harness/anomaly_judge.py`는 이
  한계를 감안해 `sep_l2`·`excl_margin`·반복 개월수(`n_months_strong`)를 주 근거로
  판정한다.
- 측정값의 절대 수치는 실제 WTMS 측정값과 무관하다(정규분포로 생성한 가짜 값).
  이 저장소는 통계 로직·LLM 판정·감사 리뷰 파이프라인의 동작을 보여주는 데
  목적이 있으며, 실제 수질 데이터를 대표하지 않는다.
